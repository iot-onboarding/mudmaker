# GitHub-facing remediation plan

Companion to [THREAT_MODEL.md](THREAT_MODEL.md). This document turns
every GitHub-related finding into a concrete engineering plan: files
to touch, wire-format changes, database migrations, tests to add, and
a rollout order that keeps the site working during the transition.

## 2. Guiding design decisions

**D-1 Move from mudurl-keyed to session-keyed authorisation.**
Today the SQLite `gitmud` table is `mudurl -> access_token`. That
makes the mudurl itself a bearer. We switch to
`session_id -> (access_token, github_login, mudurl_at_grant,
created_at, last_used_at)` where `session_id` is a 32-byte random
string generated by gitmud at OAuth completion, returned to the
browser once, and re-presented on every subsequent gitmud call as
`Authorization: Bearer <session_id>`. The mudurl is still recorded
(for auditing and to bind the session to a single MUD URL) but never
gates authorisation on its own.

**D-2 The browser stores the session in `sessionStorage`, not
`localStorage`, and not a cookie.** No cookie means no ambient
authority: an attacker's page cannot cause the browser to attach the
session to a cross-origin request. `sessionStorage` scopes it to a
single tab so a closed tab logs the user out for free.

**D-3 Same-origin only.** Since the site is same-origin (browser fetch
to `/gitShovel/*` on the same host as the HTML that made it), we add
an `Origin` allowlist server-side as defence-in-depth. Requests with
an `Origin` outside `{https://mudmaker.org, https://www.mudmaker.org}`
(or the operator's configured list) are refused with `403`.

**D-4 The GitHub API surface is treated as an untrusted string
sink.** Every path segment and query value we build is fed through
`urllib.parse.quote(..., safe="")` and every input that becomes a
segment (mfg, model, branch, filename) is passed through a
whitelist-only sanitiser that either succeeds or rejects the request.
No silent rewriting.

**D-5 Response bodies from GitHub are treated as untrusted for
logging.** Log entries strip `\r`/`\n`, truncate to 200 chars, and
`repr()` the remainder so no bytes reach the log format string
unfiltered.

**D-6 Rollout order matters more than any single fix.** Sanitisation
(P1) is deployable without any client change. Session bearer (P3)
requires a coordinated client+server rollout and must preserve the
mudurl path for one release so cached browsers keep working.

## 3. Phased delivery

| Phase | Ship-blocker | Client-side change? | DB migration? |
|-------|-------------|---------------------|---------------|
| 1. GitHub-input hardening | No | No | No |
| 2. Origin allowlist + log sanitisation | No | No | No |
| 3. Session bearer + sign-out + TTL | Yes | Yes (omud.js, tabs.js, mudmaker-reload.js) | Yes |
| 4. Retire `got_token`/`got_tok` shortcut | No | Yes (omud.js) | No |
| 5. Pin the `mudcerts` supply chain | No | No | No |

compatibility risk. Phase 3 is the one to plan around; phase 4 waits

## 4. Phase 1 — GitHub-input hardening (T-07 / T-08 / T-09 / T-10)

### 4.1 New helpers in [gitmud/gitmud/app.py](gitmud/gitmud/app.py)

Add a single module-level block near the other regexes:

```python
from urllib.parse import quote

_GIT_REF_COMPONENT_RE = re.compile(r"^[a-z0-9][a-z0-9._-]{0,63}$")
_GIT_REF_FORBIDDEN_RE = re.compile(
    r"(\.\.|@\{|//|^-|^\.|/\.|\.$|[\x00-\x1f\x7f~^:?*\[\\ ])"
)

def _sanitise_ref_component(name, field_name):
    """
    Return *name* if it is safe to embed as a single segment of a Git
    branch ref or a GitHub REST path. Otherwise raise ValueError.

    Rules:
      * NFKC-normalise
      * lowercase
      * collapse whitespace to `-`
      * reject empty, longer than 64, leading `.`/`-`, trailing `.`,
        `..`, `/`, `~`, `^`, `:`, `?`, `*`, `[`, `\\`, `@{`, `//`,
        and any control char
      * final match: ``[a-z0-9][a-z0-9._-]{0,63}``
    """
    if name is None:
        raise ValueError(f"{field_name}: missing")
    import unicodedata
    normalised = unicodedata.normalize("NFKC", str(name)).strip().lower()
    normalised = re.sub(r"\s+", "-", normalised)
    if _GIT_REF_FORBIDDEN_RE.search(normalised):
        raise ValueError(f"{field_name}: forbidden characters")
    if not _GIT_REF_COMPONENT_RE.match(normalised):
        raise ValueError(f"{field_name}: must match [a-z0-9._-]{{1,64}}")
    return normalised


def _github_path(*segments):
    """
    Compose a GitHub REST path from segments that have each already been
    validated by _sanitise_ref_component or are known-safe literals.
    Each segment is URL-quoted with no `safe` chars so `/` and `?`
    cannot escape a segment.
    """
    return "/" + "/".join(quote(s, safe="") for s in segments if s)
```

## 5. Phase 2 — Origin allowlist + log sanitisation (T-04 partial,
T-11)

### 5.1 `_safe_log()` helper

Add near the top of [gitmud/gitmud/app.py](gitmud/gitmud/app.py):

```python
_LOG_TRUNC = 200

def _safe_log(text):
    """Return *text* stripped of CR/LF/control chars, truncated,
    and quoted with repr() so it never opens a new log line."""
    if text is None:
        return ""
    trimmed = str(text)[:_LOG_TRUNC].translate(
        {c: None for c in list(range(0, 32)) + [127]}
    )
    return repr(trimmed)
```

Change every `log.warning("... %s", ..., resp.text[:200])` to
`_safe_log(resp.text)`. Sites: `git_get`, `git_putpost`, the
`log.warning("Invalid request parameters ...")` line in `/pcap2mud`,
and `log.info("stored token for %s rejected by GitHub; evicting",
mudurl)` (mudurl also comes from the request).

Test: `test_log_forgery_stripped` — pass a fake requests-Response
with `\r\nInjected: yes` in `.text`, capture log via `caplog`, assert
the line is a single line and contains `\\r\\n` literally.

### 5.2 `Origin` allowlist

Add a `before_request` hook:

```python
_ALLOWED_ORIGINS = set(
    o.strip() for o in _CONFIG["security"]
      .get("allowed_origins",
           "https://mudmaker.org,https://www.mudmaker.org")
      .split(",") if o.strip()
)

@app.before_request
def _origin_guard():
    if request.method in ("GET", "HEAD", "OPTIONS"):
        return None
    origin = request.headers.get("Origin")
    if origin is None:
        # curl / non-browser clients omit Origin. Accept only when
        # there is also no Referer with a foreign host.
        referer = request.headers.get("Referer")
        if referer:
            from urllib.parse import urlparse
            u = urlparse(referer)
            check = f"{u.scheme}://{u.netloc}"
            if check not in _ALLOWED_ORIGINS:
                return jsonify({"error": "cross-origin refused"}), 403
        return None
    if origin not in _ALLOWED_ORIGINS:
        return jsonify({"error": "cross-origin refused"}), 403
    return None
```

Config gains a `[security] allowed_origins = ...` list. Both
[gitmud/config.ini](gitmud/config.ini) and the tests fixture supply a
value. `/pcap2mud` and `/gottoken` behave identically — this is
gitmud-wide.

Tests:
- `test_cross_origin_post_refused` — POST `/therest` with
  `Origin: https://evil.example`; expect 403 and no GitHub call.
- `test_same_origin_post_allowed` — POST with the configured origin;
  passes existing test.
- `test_no_origin_no_referer_allowed` — curl-style call; passes.

### 6.2 New auth helpers in
[gitmud/gitmud/app.py](gitmud/gitmud/app.py)

```python
import secrets, time
from functools import wraps

SESSION_TTL_SECONDS = int(_CONFIG["security"].get(
    "session_ttl_seconds", 90 * 24 * 3600))

def _new_session(github_resp, github_login, mudurl):
    session_id = secrets.token_urlsafe(32)
    now = int(time.time())
    con = sqlite3.connect(MUD_DB)
    with con:
        con.execute(
            "INSERT INTO sessions VALUES (?,?,?,?,?,?,?,?)",
            (session_id, github_login,
             github_resp["access_token"], mudurl,
             github_resp.get("scope"),
             github_resp.get("token_type"),
             now, now))
    return session_id

def _resolve_session(bearer):
    if not bearer:
        return None
    con = sqlite3.connect(MUD_DB)
    row = con.execute(
        "SELECT session_id, github_login, access_token, mudurl, "
        "created_at FROM sessions WHERE session_id = ?",
        (bearer,)).fetchone()
    if not row:
        return None
    session_id, login, token, mudurl, created = row
    if int(time.time()) - created > SESSION_TTL_SECONDS:
        con.execute("DELETE FROM sessions WHERE session_id = ?",
                    (session_id,))
        con.commit()
        return None
    con.execute("UPDATE sessions SET last_used_at = ? "
                "WHERE session_id = ?", (int(time.time()), session_id))
    con.commit()
    return {"session_id": session_id, "login": login,
            "token": token, "mudurl": mudurl}

def _bearer_from_request():
    header = request.headers.get("Authorization", "")
    if header.startswith("Bearer "):
        return header[7:].strip() or None
    return None

def requires_session(fn):
    @wraps(fn)
    def wrapper(*a, **kw):
        sess = _resolve_session(_bearer_from_request())
        if not sess:
            return jsonify({"error": "not authenticated"}), 401
        request.session = sess
        return fn(*a, **kw)
    return wrapper
```

### 6.3 Route changes

- `POST /oAuthv2` — unchanged input schema, but on success returns
  `{"user": login, "session": "<session_id>"}` and does NOT store a
  `mudurl → token` row. Client stores the session in
  `sessionStorage`.
- `POST /dorepo`, `POST /branch`, `POST /therest` — add
  `@requires_session`. Read `user` from `request.session["login"]`,
  not the request body. Remove the `token_user != user` cross-check
  (now unreachable). Kills T-01, T-06.
- `GET /gottoken` — **removed**. Client uses local `sessionStorage`
  to know whether it has a bearer. Kills T-02.
- New `POST /signout` — reads bearer, calls GitHub
  `DELETE /applications/{client_id}/token` with HTTP Basic
  (`client_id:client_secret`) and body `{"access_token": <token>}`
  to revoke the OAuth grant upstream, then deletes the row locally.
  Kills T-26. Returns `204`.
- New `GET /whoami` — returns `{"login": ...}` for the current
  session, `401` otherwise. Used by the UI to render the "signed in
  as X" pill and the sign-out button.

### 6.4 Background sweeper (T-18)

Add a lightweight janitor that runs on app import:

```python
import threading
def _janitor_loop():
    while True:
        try:
            cutoff = int(time.time()) - SESSION_TTL_SECONDS
            con = sqlite3.connect(MUD_DB)
            with con:
                con.execute("DELETE FROM sessions WHERE created_at < ?",
                            (cutoff,))
        except Exception:
            log.exception("sessions janitor failed")
        time.sleep(3600)

_JANITOR = threading.Thread(target=_janitor_loop, daemon=True)
_JANITOR.start()
```

(One thread per gunicorn worker is fine — the `DELETE` is idempotent.)

## 7. Phase 4 — Retire `got_token`/`got_tok` (T-03)

The shortcut `jsonbody["got_token"] = true` in
[assets/js/omud.js:169](assets/js/omud.js#L169) is what makes T-01
exploitable in the first place (it tells the server "reuse the
mudurl-keyed token without a fresh code"). Phase 3 already removes
the server-side lookup path when a bearer is present.

Steps:
1. In Phase 3.1, add `log.warning("legacy got_token/got_tok path
   used from origin=%s ua=%s", ...)` inside `complete_oauth` when
   the branch fires.
2. After Phase 3 has been in production long enough for
   `sessionStorage`-holding browsers to dominate (rule of thumb:
   two weeks + one release), delete the branch:
   ```python
   if "got_token" in req or "got_tok" in req:
       return jsonify({"error": "legacy path removed; please reload"}), 410
   ```
3. Delete the client code that ever sets it
   ([omud.js:112–118, 168–169](assets/js/omud.js#L112)).
4. Delete `sessionStorage.setItem("gottoken","true")` in
   [tabs.js](assets/js/tabs.js#L165) and the corresponding cleanup
   in [mudmaker-reload.js](assets/js/mudmaker-reload.js#L61).

## 8. Phase 5 — Pin the `mudcerts` supply chain (T-28)

[Dockerfile](Dockerfile#L1) currently:

```dockerfile
ARG MUDCERTS_REF=main
RUN git clone https://github.com/iot-onboarding/mudcerts.git /src/mudcerts \
    && cd /src/mudcerts && git checkout "${MUDCERTS_REF}"
```

Change to:

```dockerfile
# Pin to a specific commit SHA. Bump this on each mudcerts release.
ARG MUDCERTS_REF=<40-char sha>
ARG MUDCERTS_EXPECTED_SHA=${MUDCERTS_REF}
RUN git clone https://github.com/iot-onboarding/mudcerts.git /src/mudcerts \
    && cd /src/mudcerts \
    && git checkout "${MUDCERTS_REF}" \
    && [ "$(git rev-parse HEAD)" = "${MUDCERTS_EXPECTED_SHA}" ] \
    && go mod download \
    && go mod verify \
    && CGO_ENABLED=0 go build -trimpath -ldflags="-s -w" \
       -o /out/mudzipserver ./web
```

Add a CI job (`.github/workflows/ci.yml`) that:
- Fails the build if `MUDCERTS_REF` looks like a branch name (not
  40 hex chars).
- Runs `go mod verify` on the built image contents.
- (Optional, later) Verifies a signed tag or a cosign attestation
  against the mudcerts release.

The bump procedure becomes a two-line PR here whenever mudcerts
releases: update `MUDCERTS_REF` and `MUDCERTS_EXPECTED_SHA` in
lockstep. Store the current pin in
`/memories/repo/deployment.md` so operators know what shipped.
